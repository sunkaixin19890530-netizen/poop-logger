## 三、社交属性设计（纯前端分享版）

### 3.1 分享卡片生成
一键生成精美的分享图片，包含：
- 🏆 你的成就展示
- 📊 你的数据统计
- 🎨 个性化装饰
- 🔗 带加密数据的分享链接（可选）

### 3.2 分享链接（核心功能）
生成特殊的分享链接，包含：
- **成绩分享**：分享你的最佳成绩
- **成就分享**：分享你解锁的成就
- **月度报告**：分享你的月度拉屎总结
- **挑战邀请**：发起挑战，朋友打开链接可以看到你的成绩并接受挑战

### 3.3 挑战系统（基于链接）
- **发起挑战**：生成挑战链接，设置挑战目标（比如"连续7天完美香蕉"）
- **接受挑战**：朋友打开链接，导入挑战目标到自己的本地应用
- **分享结果**：完成挑战后生成结果卡片分享

### 3.4 「屎命召唤」PK 好友系统（无需后端）

#### 3.4.1 核心概念
- **以屎会友**：好友关系通过「战斗链接」建立，而非传统好友码
- **本地对战**：所有 PK 数据存储在本地，无网络需求
- **战绩永恒**：PK 历史记录永久保存，成就「屎」刻铭记

#### 3.4.2 好友系统重构
- **添加好友**：通过接受对方的「战斗链接」自动成为好友
- **好友数据**：每次 PK 后同步对方最新战绩（存储在本地）
- **本地排行榜**：你和好友的「屎力」排名
- **PK 历史**：记录每一场史诗级对决

#### 3.4.3 「屎命召唤」PK 玩法
1. **发起挑战**：
   - 生成带数据的「战斗链接」（或二维码）
   - 包含：最近7天战绩、挑衅语录、数据哈希值（防作弊）
   - 可选时效：24小时内未接受自动失效

2. **接受挑战**：
   - 好友点击链接/扫码自动导入数据
   - 本地 PK 引擎自动计算结果
   - 生成「战报」卡片

3. **PK 维度**：
   - 📊 数值对决：平均得分、连续天数、总记录数
   - 🎨 形态相克：根据布里斯托分类法设计属性克制
   - 🎰 随机事件：便秘暴击、通畅加成等趣味变量

4. **战利品系统**：
   - 专属「便便勋章」（如「碾压者」「运气之王」）
   - 解锁特殊「排便皮肤」
   - 对方的「黑历史」数据碎片（可拼接成趣味图鉴）

#### 3.4.5 分享链接格式
```
poop://pk?data=H4sIAAAAAAAA...
```
- `data`：base64 编码的 JSON 数据，包含：
  - `type`: 分享类型（pk_challenge/pk_result/friend_add）
  - `payload`: 具体数据内容
  - `timestamp`: 时间戳
  - `signature`: 数据签名（防篡改）

### 3.5 分享方式
- 📸 图片分享：生成PNG图片分享到微信、朋友圈等
- 🔗 链接分享：分享加密链接，好友打开可查看
- 📱 二维码分享：生成二维码，扫码即可查看

## 四、技术架构（纯前端）

### 4.1 技术栈
- **框架**：Vue 3 + Vite（轻量快速）
- **UI**：Element Plus 或 Naive UI
- **图表**：ECharts
- **存储**：IndexedDB + localStorage
- **图片生成**：html2canvas（生成分享图片）
- **PWA**：支持离线使用和添加到主屏幕

### 4.2 数据存储方案
- **本地存储**：所有个人数据存储在 IndexedDB 中
- **分享数据**：通过 URL 参数传递加密后的数据（base64 编码）
- **好友数据**：好友分享的数据存储在本地 IndexedDB 中

### 4.3 分享链接格式
```
https://your-domain.com/share?data=eyJ0eXBlIjoiY...
```
- `data`：base64 编码的 JSON 数据
- 包含分享类型、数据内容、时间戳等

### 4.4 数据加密（可选）
- 使用简单的加密算法对分享数据进行编码
- 防止数据被轻易篡改
- 可选：设置分享密码保护隐私
